01 The problem

Every export is a dead end until someone with time and SQL shows up.

Data teams ship dumps and CSVs constantly — CRM exports, admissions pipelines, order histories. Almost none of it gets looked at twice.

  • × Building a real dashboard means a BI person, a ticket, and a few days.
  • × Asking one follow-up question means finding whoever knows SQL.
  • × Multi-file exports (leads + applications, customers + orders) usually never get joined at all — they just sit as separate files.
02 The idea

Upload the file. The AI does the analyst's job.

Not a chatbot bolted onto a spreadsheet — SQLens actually reads your schema and relationships, decides what's worth showing, writes the SQL itself, and keeps a live conversation open once the dashboard exists.

03 How it works

Upload to dashboard in under a minute.

04 What's built

Not a demo trick — the whole loop is real.

Multi-file ingestion

SQL, CSV, and Excel in one upload, loaded into a shared database so cross-file joins just work.

AI-designed dashboard

A bento-style layout sized to what each chart actually needs, plus a written key insight.

Live chat, real SQL

Streams over WebSocket, shows the query it ran, retries cleanly on failure.

Knowledge injection

Tell it what the columns actually mean — it steers both the dashboard and the chat.

Per-chart controls

Refresh, expand, remove, or export any chart as a PNG or CSV.

Shareable report

One click exports the dashboard, chat transcript, and every SQL query as markdown.

06 Safety & cost

Built assuming someone will try to break it.

GUARDRAILS
Every AI-written query runs on a read-only connection, SELECT-only, single-statement, row-capped, and timed out — enforced by SQLite itself, not just application logic.
RATE LIMITS
Capped per session and per IP — a fresh upload can't be used to reset a session's quota.
HARD BUDGET
A real, persisted dollar ceiling across every user — once hit, every AI call stops until it's raised, and a restart can't quietly reset the clock.
RESILIENCE
Sessions survive a backend restart; transient AI failures retry automatically; the rest surface as a clear message with a one-click retry.
